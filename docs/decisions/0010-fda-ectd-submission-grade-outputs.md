# ADR-0010: FDA eCTD 3.2.2 Module 5 Submission-Grade Outputs

**Date:** 2026-06-10
**Status:** Accepted
**Authors:** backend-agent, product-owner
**Reviewers:** architect-agent (pending), audit-compliance-agent (pending)

## Context

Phase 1 delivered a **Demonstrable** submission packaging workflow (readiness → create → manifest → ZIP) on synthetic data, with honest placeholder labeling for CSR PDF and SDRG. The product target is **FDA US eCTD 3.2.2 Module 5** submission-readiness:

- SDTM/ADaM datasets as **SAS XPT** (define.xml already references `.xpt` hrefs)
- **ICH E3** CSR PDF (sections 1, 2, 9–15)
- **SDRG** and **ADRG** review documents
- **eCTD backbone** (`index.xml`, `index-md5.txt`)
- Internal CDISC validation gates (Pinnacle 21 when licensed via #11)

Parallel development: issues **#10–#15, #17** are owned by another engineer. Lane 0–2 services must not modify delegated files.

---

## Decision

### 1. Dataset transport

- Submission ZIP uses **SAS XPT v5** files produced by `xpt_export_service.py` (pyreadstat).
- CSV remains a **dev/export** format only; not included in FDA submission packages (Lane 3 integration).

### 2. SDTM Implementation Guide

- **SDTM IG 3.3** per ADR-0008.

### 3. CSR PDF

- **ReportLab** via dedicated `csr_ich_e3_pdf_renderer.py` (not generic `export_pdf()`).
- **ICH E3 structure:** sections 1, 2, 9–15; synopsis as structured table.
- **TLF policy:** CSR PDF **references** TLF table IDs; full tables remain in separate `tlf/*.rtf` files (FDA-standard separation).
- CSR **prose** required in artifact content — produced by **#17**; packaging gate enforced in Lane 3.

### 4. Review documents

- **SDRG** generated by `reviewers_guide_generator.py` (Lane 3); download endpoint on **`submissions.py`** (not `csr.py`) to avoid #17 merge conflicts.
- **ADRG** produced by **#13** (`adrg_generation_service.py`).

### 5. eCTD backbone

- `ectd_xml_export_service.py` generates `index.xml` + `index-md5.txt` from package manifest (eCTD 3.2.2 structure).
- Additive endpoint: `GET /api/v1/submissions/{package_id}/ectd-xml`.

### 6. Validation

- **Internal CDISC engine** is authoritative until Pinnacle 21 is licensed (#11).
- `define_xml_validator.py` validates define.xml structure and XPT href alignment.
- Submission readiness (Lane 3) requires zero FAIL `ValidationEvidence` + define XSD/structure pass + XPT/define cross-check.

### 7. FDA Module 5 folder layout (Lane 3 `submission_package_builder.py`)

```
m5/datasets/tabulation/sdtm/*.xpt
m5/datasets/tabulation/sdtm/define.xml
m5/datasets/tabulation/sdtm/sdrg.pdf
m5/datasets/analysis/adam/*.xpt
m5/datasets/analysis/adam/adam-define.xml
m5/datasets/analysis/adam/adrg.pdf
m5/clinical-study-reports/{csr_id}.pdf
tlf/{artifact_id}.rtf
index.xml
index-md5.txt
```

### 8. Data classification

| Value | Meaning |
|-------|---------|
| `SYNTHETIC_CONFORMANCE` | Synthetic data; CDISC rules pass; not for regulatory filing |
| `REGULATORY_DRAFT` | Real or synthetic with full package; pending final QC |
| `PRODUCTION` | Operator-approved for filing (future; requires BAA/DPA for PHI) |

## Consequences

### Positive

- Clear separation between parallel lanes (no merge conflicts with #10–#17).
- define.xml ↔ XPT contract enforced before packaging.
- Dedicated CSR renderer enables ICH E3 layout without polluting generic export.

### Negative

- Lane 3 integration blocked until #13 and #17 merge.
- Full CDISC Define-XML XSD validation requires official schema assets (structural validation in v1).
- XPT type coercion may lose precision for edge-case clinical variables.

### Neutral

- `submission_package_builder.py` (Lane 3) will absorb assembly logic from `submission_service.py`.

---

## Alternatives Considered

### Alternative A: Embed TLF tables inside CSR PDF

**Why rejected:** High layout risk; FDA typically receives TLFs as separate outputs referenced from CSR.

### Alternative B: CSR PDF via DOCX → PDF conversion

**Why rejected:** ReportLab already in stack; direct PDF gives better control for ICH E3 page structure.

### Alternative C: Keep CSV in submission ZIP

**Why rejected:** define.xml hrefs reference `.xpt`; CSV breaks CDISC submission contract.
